# Redis Setup

Redis powers BullMQ, handling all ingestion, normalization, and deduplication queues for Eventio.

## Local Setup
Use Docker for local development:
```bash
docker run -d --name eventio-redis -p 6379:6379 redis:alpine
```
Set `.env`:
```env
REDIS_URL="redis://localhost:6379"
```

## Production Setup (Railway / Upstash)

### Option A: Railway Redis (Recommended)
Since the worker processes run on Railway, deploying Redis in the same internal network guarantees single-digit millisecond latency.
1. Go to your Railway project.
2. Click **New** -> **Database** -> **Add Redis**.
3. Use the `REDIS_PRIVATE_URL` generated by Railway as the `REDIS_URL` in your worker environment variables.

### Option B: Upstash (Serverless Redis)
If deploying workers elsewhere, Upstash provides a serverless Redis offering.
1. Create a database on [Upstash](https://upstash.com/).
2. Copy the `redis://...` connection string.
3. Add to your `.env`.

## BullMQ Configuration Requirements
Ensure `maxmemory-policy` is set to `noeviction`. BullMQ relies on persistent queue structures; eviction can lead to corrupted or lost jobs.
