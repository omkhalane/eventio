# SEO and Discoverability

## GitHub SEO

Recommended repository description:

> Open-source developer event aggregator for coding contests, hackathons,
> AI competitions, tech conferences, and hiring challenges.

Canonical repository:

```text
https://github.com/omkhalane/eventio
```

Recommended topics:

```text
developer-events, hackathons, coding-contests, competitive-programming,
calendar, ai-competitions, tech-events, react, typescript, python, scraper
```

README should keep keyword-rich headings without stuffing:

- Developer event aggregator
- Hackathon calendar
- Coding contest calendar
- Competitive programming events
- AI competitions and tech conferences

## Web SEO

The app should keep:

- canonical URL
- concise title and description
- Open Graph image
- Twitter card metadata
- JSON-LD for `WebSite` and `SoftwareApplication`
- sitemap.xml
- robots.txt

Future dynamic pages should add semantic routes:

- `/hackathons`
- `/coding-contests`
- `/ai-competitions`
- `/tech-conferences`
- `/events/:platform`

## Content Strategy

Landing page sections should answer:

- What events are tracked?
- Which platforms are indexed?
- How often is data refreshed?
- How do developers filter by topic, date, and platform?
- How can contributors add sources?

Primary keyword targets:

- developer events calendar
- hackathon calendar
- coding contest calendar
- competitive programming contests
- AI competition calendar
- tech conference aggregator
