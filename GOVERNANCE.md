# Governance

Eventio is currently maintained by [Om Khalane](https://github.com/omkhalane).

## Project Values

- Useful developer discovery over noisy feature sprawl.
- Reliable data handling over clever one-off scrapers.
- Clear docs over hidden maintainer knowledge.
- Practical architecture over unnecessary complexity.
- Respectful collaboration over rushed debate.

## Maintainer Responsibilities

The maintainer is responsible for:

- Reviewing issues and pull requests.
- Setting project direction and scope.
- Protecting user privacy and repository security.
- Keeping architecture and contribution standards coherent.
- Maintaining release notes, docs, and dependency hygiene.
- Deciding when a source, feature, or integration belongs in the project.

## Decision Making

The maintainer has final decision authority on:

- Roadmap and priorities.
- Architecture changes.
- Database schema direction.
- Scraper source acceptance.
- Release timing.
- Security disclosure handling.

Contributors are encouraged to propose changes through focused issues and pull requests with clear tradeoffs.

## Contribution Expectations

Good contributions are:

- Focused.
- Tested.
- Documented.
- Respectful of existing patterns.
- Clear about risk and rollout.

Large rewrites should start with an issue or design note before implementation.
